Backup & Restore Guide

How to back up and restore your n8watch database, configuration, and logs using the optional backup.sh script.

Supplementary Guide · Version 1.0 · Node.js 24 LTS

1 · Overview

n8watch ships an optional backup.sh shell script that archives all critical data into a timestamped .tar.gz file. The archive includes the SQLite database, the active config.yaml, and application log files. Backups can be restored to any compatible n8watch installation.

What Is Backed Up

ItemPathNotes
SQLite databasedata/n8watch.dbAll targets, ping results, and alert records
WAL sidecardata/n8watch.db-walIncluded when present (uncommitted write-ahead log)
SHM sidecardata/n8watch.db-shmIncluded when present (shared-memory index)
Configurationconfig.yamlYour current running configuration file
Log fileslogs/out.log and error.log
Tip: Run backups on a schedule (e.g. via cron) and store archives on a separate host or cloud storage to protect against data loss.

2 · Usage

Command Synopsis

#
# n8watch Backup / Restore Script
#
# Usage:
#   ./backup.sh --backup
#   ./backup.sh --backup --dry-run
#   ./backup.sh --backup --dry-run --output /path
#   ./backup.sh --restore /path/file.tar.gz
#   ./backup.sh --help
#

2.1 Create a Backup

Create a compressed archive in the default ./backups/ directory:

./backup.sh --backup

The archive is named with a UTC timestamp, for example: n8watch-backup-2025-01-15T12-00-00.tar.gz

2.2 Dry Run

Preview what would be included in the backup without actually creating the archive. Useful for verifying paths before committing to a backup. 

# Dry-run to default output location
./backup.sh --backup --dry-run

# Dry-run with a custom output directory
./backup.sh --backup --dry-run --output /mnt/nas/backups

2.3 Custom Output Directory

Save the archive to a directory of your choice:

./backup.sh --backup --output /mnt/nas/backups

2.4 Restore from Archive

Stop the application before restoring. Restoring while n8watch is running can corrupt the database. Use npm run pm2:stop or Ctrl+C. 
# Stop n8watch
npm run pm2:stop

# Restore from a backup archive
./backup.sh --restore /path/to/n8watch-backup-2025-01-15T12-00-00.tar.gz

# Restart n8watch
npm run pm2:start

2.5 Help

./backup.sh --help

3 · Flags Reference

FlagArgumentDescription
--backup — Create a compressed .tar.gz archive of the database, config, and logs
--restore <file> Restore database, config, and logs from the specified .tar.gz archive. Overwrites existing files.
--dry-run — Show what would be included in the backup without creating the archive. Must be combined with --backup.
--output <dir> Directory where the backup archive is written. Defaults to ./backups/ inside the n8watch installation directory. Created automatically if it does not exist.
--help — Print usage information and exit

Flag Combinations

4 · Restore Procedure (Step by Step)

  1. Stop the application — prevent database corruption during restore: 
    npm run pm2:stop
# or if running directly: Ctrl+C
  2. Optionally back up your current state before overwriting: 
    ./backup.sh --backup --output /tmp/pre-restore-snapshot
  3. Run the restore: 
    ./backup.sh --restore /path/to/n8watch-backup-2025-01-15T12-00-00.tar.gz

    The script extracts the archive and places the database into data/, the config into the project root, and logs into logs/.

  4. Verify the restored config: 
    cat config.yaml

    Confirm that targets, SMTP settings, and interface definitions look correct.

  5. Restart n8watch: 
    npm run pm2:start
Warning: Restore overwrites data/n8watch.db and config.yaml in place. There is no automatic rollback. Always take a pre-restore snapshot (step 2 above) if your current data has value.

Scheduled Backups with cron

To back up automatically every night at 02:00, add the following line to your crontab (crontab -e): 

0 2 * * * /opt/n8watch/backup.sh --backup --output /mnt/nas/n8watch-backups >> /var/log/n8watch-backup.log 2>&1
Adjust the path (/opt/n8watch/) to match your installation directory. Ensure the output directory is writable by the user running the cron job.
